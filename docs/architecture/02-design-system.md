# Volume 02: Design System

## Purpose

Document the visual and interaction rules that keep Downtown Perks polished, calm, and consistent.

## Tokens

- Background: `#F7F8FB`
- Surface: `#FFFFFF`
- Navy: `#0B1F33`
- Secondary navy: `#132238`
- Gold: `#C8A96A`
- Deep gold: `#B8963E`
- Border: `rgba(11,31,51,0.08)`
- UI typography: Inter
- Display typography: Instrument Serif for major editorial headings only

## Layout Rules

- Mobile first.
- Left aligned by default.
- White panels with thin borders.
- Rectangular premium layout.
- Horizontal scroll for dense tables and rails.
- No clipped panels, drawers, controls, or tables.
- Avoid nested card stacks.

## Component Standards

- Buttons must perform a real action or route to a real page.
- Tables must support readable wrapping or horizontal scrolling.
- Forms require labels, validation, loading state, success state, and error state.
- Modals and drawers require escape close, focus management, and viewport-safe scrolling.

## Copy Rules

Use everyday language. Avoid jargon such as "signal", "movement", "unlock", "frictionless", and generic SaaS hype unless context requires it.

## Accessibility

Minimum target: WCAG AA contrast, visible focus states, keyboard navigation, semantic labels, and logical tab order.
