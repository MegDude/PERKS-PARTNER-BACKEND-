# Volume 06: Partner Platform

## Purpose

Define the workspace partners use to manage offers, events, campaigns, reports, QR codes, billing, contacts, files, and team access.

## Core Modules

- Workspace home
- Offers
- Events
- Campaigns
- Reports
- Calendar
- Intelligence
- Pricing
- Settings

## Partner Workspace Standard

Every module must support clear empty state, loading state, actionable CTA, validation, persistence, audit logging, and responsive mobile layout.

## Lifecycle

Register -> price -> checkout -> provision -> setup -> publish -> report -> renew.

## Data Relationships

Partner workspaces reference registry entities rather than duplicating place, campaign, event, and offer data.

## Required CTAs

All partner-facing CTAs must route to real flows: registration, pricing, checkout, workspace, campaign builder, map, support, or settings.
