# Volume 07: Resident Platform

## Purpose

Define the resident-facing experience for discovery, perks, events, saved places, recommendations, and community participation.

## Core Surfaces

- Map
- Perks
- Events
- Saved places
- Search
- Ask the Map
- Resident card
- Preferences
- Notifications
- Surveys

## Map Rule

The resident map must use real map and registry data. Static embeds or placeholder map canvases are not acceptable for map routes.

## Personalization

Recommendations should use registry data, user preferences, location context, campaign eligibility, and saved interactions.
